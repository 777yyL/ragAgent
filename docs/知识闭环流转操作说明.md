# 知识闭环流转操作说明

> 发电知识中心 · 知识全生命周期操作指南
> 基于 RAGFlow HTTP API（13 大模块、80+ 接口）落地
> 版本：v1.0  日期：2026-06-17

---

## 一、闭环总览

知识在系统中以 **8 个阶段** 顺次流转，形成"采集 → 加工 → 入库 → 标注 → 审核 → 检索 → 应用 → 运营"的完整闭环；运营阶段的反馈（命中统计、用户反馈、审核退回）反向驱动新一轮的知识迭代采集。

```
┌─────────────────────────────────────────────────────────────┐
│                     知识闭环生命周期                          │
└─────────────────────────────────────────────────────────────┘
   ①采集      ②加工       ③入库        ④标注       ⑤审核
 ┌──────┐   ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
 │上传  │ → │解析  │ →  │向量化│ →  │AI自动│ →  │AI预审│
 │数据源│   │切分  │    │图谱  │    │标注  │    │人工审│
 └──────┘   └──────┘    │RAPTOR│    │人工补│    │发布  │
                          └──────┘    └──────┘    └───┬──┘
                                                      │
   ⑧运营      ⑦应用       ⑥检索                       │
 ┌──────┐    ┌──────┐    ┌──────┐                      │
 │权限  │ ←│智能体│ ← │语义  │ ←────────────────────────┘
 │版本  │    │对话  │    │检索  │
 │统计  │    │溯源  │    │过滤  │
 └──┬───┘    └──────┘    └──────┘
    │ 反馈（命中/退回/缺口）驱动迭代
    └──────────────────────────────────────────────────────→ 回到 ①
```

**收敛原则**：所有阶段产出的文档/切片最终都收敛到 RAGFlow 的 Dataset/Document/Chunk 三层模型，自建后端只做编排、权限策略、审核状态机，不另起存储。

## 三、分阶段操作说明

### 阶段 ① 知识采集与抽取

**目标**：把外部异构知识源接入系统。**操作页**：`upload-wizard.html`（引导式上传）/ `documents.html`（日常管理）/ `data-source.html`（二期数据源）。

| 操作 | 页面 | 动作 | RAGFlow API |
|------|------|------|------------|
| 首次上传（向导） | upload-wizard 第①步 | 选择文件上传到指定知识库 | `POST /api/v1/datasets/{dataset_id}/documents` |
| 日常批量上传 | documents | 点击"上传"按钮，拖拽文件 | 同上 |
| 数据库/API/MQ 接入（二期）| data-source | 配置 Connector，输出 JSON 后调上传接口 | 收敛到 Upload documents |
| 查看上传记录 | documents | 文档列表查看大小/上传人/MD5 | `GET /api/v1/datasets/{dataset_id}/documents` |

**注意**：上传前必须先在「知识库管理」创建目标 dataset；上传后文档默认进入"待解析"状态。

---

### 阶段 ② 知识处理与加工

**目标**：把原始文件解析、清洗、切分为可检索的 chunk。**配置页**：`dataset.html`（chunk_method + parser_config 在创建/编辑知识库时设置）；**执行页**：`documents.html`（触发解析）；**查看页**：`detail.html`。

| 操作 | 页面 | 动作 | RAGFlow API |
|------|------|------|------------|
| 配置切分方法 | dataset（新建/编辑弹窗）| 12 种 chunk_method 选一（naive/qa/laws/book/table/picture...）| `POST/PUT /api/v1/datasets` body:`chunk_method` |
| 配置解析参数 | dataset 弹窗 | chunk_token_num / delimiter / auto_keywords / auto_questions / DeepDOC / html4excel | `parser_config` 字段 |
| 触发解析 | documents | 选中文档 → "重新解析" | `POST /api/v1/datasets/{dataset_id}/chunks`（Parse documents）|
| 停止解析 | documents | "停止"按钮 | `POST /api/v1/datasets/{dataset_id}/chunks`（Stop parsing，传 stop=true）|
| 查看切片结果 | detail | 进入文档详情，浏览 chunk 列表 | `GET .../documents/{document_id}/chunks`（List chunks）|

**关键**：DeepDOC 引擎自动完成 OCR、表格结构识别（TSR）、版面分析；`auto_keywords`/`auto_questions` 在解析时自动生成关键词与问题，无需额外标注动作。

---

### 阶段 ③ 知识入库

**目标**：把切片向量化、构建图谱/RAPTOR、写入权限元数据。**配置页**：`dataset.html`；**可视化页**：`knowledge-graph.html`。

| 操作 | 页面 | 动作 | RAGFlow API |
|------|------|------|------------|
| 选择 Embedding 模型 | dataset 弹窗 | 创建时指定 `embedding_model`（如 bge-large-zh-v1.5@BAAI）| `POST /api/v1/datasets` body:`embedding_model` |
| 启用 GraphRAG | dataset 弹窗"高级增强" | 开启 `parser_config.graphrag.use_graphrag` | 创建/更新 dataset |
| 构建知识图谱 | knowledge-graph | "构建图谱"按钮 | `POST /api/v1/datasets/{dataset_id}/knowledge_graph`（Construct）|
| 查询图谱构建状态 | knowledge-graph | 自动轮询 | `GET .../knowledge_graph/status` |
| 启用 RAPTOR 多跳 | dataset 弹窗 | 开启 `parser_config.raptor.use_raptor` | 创建/更新 dataset |
| 构建 RAPTOR | dataset 卡片"构建 RAPTOR" | 触发 | `POST /api/v1/datasets/{dataset_id}/raptor`（Construct）|
| 启用父子分块 | dataset 弹窗 | 开启 `parent_child.use_parent_child` | 创建/更新 dataset |
| 写入权限/分类元数据 | permission 或后端自动 | 把 dept_ids/categories/security_level 写入文档 metadata | `POST /api/v1/datasets/{dataset_id}/metadata/update`（批量）|

**关键**：权限元数据必须在此阶段写入（否则阶段⑥的检索预过滤无法生效）。

---

### 阶段 ④ 知识标注

**目标**：补全关键词、问题、标签以提升召回率与准确度。**操作页**：`annotation.html`。**机制**：AI 自动标注（解析时已生成）+ 人工校验补充。

| 操作 | 页面 | 动作 | RAGFlow API |
|------|------|------|------------|
| 查看 AI 自动标注 | annotation 左侧 chunk 列表 | 切到"已标注"看 auto_keywords/auto_questions 结果 | List chunks 返回的 `important_keywords`/`questions` |
| 修改/补充关键词 | annotation 右侧编辑区 | 编辑关键词云、新增/删除 | `PATCH .../chunks/{chunk_id}` body:`important_keywords` |
| 补充预期问题 | annotation 编辑区 | 添加 Q&A（用于问答增强）| `PATCH .../chunks/{chunk_id}` body:`questions` |
| 设置标签 | annotation 编辑区 | 添加 tag_kwd | `PATCH .../chunks/{chunk_id}` body:`tag_kwd` |
| 设置重要度 | annotation 滑块 | 影响检索权重 | 业务侧元数据，可写入 chunk metadata |
| 文档级标签批量更新 | permission/documents | 批量打标签 | `POST /api/v1/datasets/{dataset_id}/metadata/update` |

**关键**：标注完成后，chunk 仍处于 `available=false`（待审），不会参与检索。

---

### 阶段 ⑤ 知识审核

**目标**：三级审核通过后正式发布到检索。**操作页**：`audit-workbench.html`（人工）/ `ai-audit-report.html`（AI）。

#### 5.1 AI 预审（异步，不阻塞流程）

| 操作 | 页面 | 动作 | 实现 |
|------|------|------|------|
| 查看 AI 6 类检测 | ai-audit-report | 浏览矛盾/错误/时效/完整/一致/规范问题 | 自建 LLM 编排 + 调 `Retrieve chunks` 找相似 |
| 采纳/忽略建议 | ai-audit-report | 批量操作 | 自建记录 |
| 定位原文 | ai-audit-report | 点击跳转 chunk | 跳 detail.html |

#### 5.2 人工三级审核（核心）

| 操作 | 页面 | 动作 | 实现 |
|------|------|------|------|
| 选择审核模板 | upload-wizard 第⑤步 / 后台配置 | 一级直审 / 两级 / 三级集团审核 | 自建模板配置（不引入 Flowable）|
| 提交审核 | upload-wizard 第⑤步 | 进入企业初审节点 | 自建状态机：DRAFT→PENDING |
| 一审（企业）| audit-workbench | 查看 chunk + AI 报告 → 通过/退回 | 自建流转 |
| 二审（区域）| audit-workbench | 同上 | 自建流转 |
| 三审（集团）| audit-workbench | 同上（会签可选）| 自建流转 |
| 终审通过 → **发布** | audit-workbench"通过"按钮 | 启用 chunk 参与检索 | **`PATCH /api/v1/datasets/{dataset_id}/documents/{document_id}/chunks`**（Update chunk availability，`available=true`）|
| 退回修改 | audit-workbench"退回" | 回到编辑员草稿 | 自建状态机：PENDING→DRAFT |
| 删除 | audit-workbench/documents | 彻底删除 | `DELETE .../chunks` 或 `DELETE .../documents` |

**关键**：`Update chunk availability` 是发布动作的唯一 API 入口；退回不删除数据，仅维持 `available=false`。

---

### 阶段 ⑥ 知识检索

**目标**：员工按权限检索已发布知识。**操作页**：`search.html`（日常）/ `retrieval-test.html`（调试）。

| 操作 | 页面 | 动作 | RAGFlow API |
|------|------|------|------------|
| 关键词/语义检索 | search | 输入查询 | `POST /api/v1/retrieval` body:`question`/`dataset_ids` |
| 权限预过滤（自动）| 后端透明 | 按用户身份拼 metadata_condition | 同上 body:`metadata_condition`（dept/security_level 过滤）|
| 图谱多跳召回 | search 高级选项 | 开启 use_kg | 同上 body:`use_kg=true` |
| 目录增强 | search 高级选项 | 开启 toc_enhance | 同上 body:`toc_enhance=true` |
| 调权重/阈值 | retrieval-test | 调 vector_similarity_weight / similarity_threshold / top_k | 同上 |
| 切换 rerank | retrieval-test | 选 rerank_id | 同上 body:`rerank_id` |
| 高亮 / 关键词匹配 | 后台默认 | 开启 highlight/keyword | 同上 body:`highlight=true`/`keyword=true` |
| 查看溯源 | search 结果 | 点击"原文"跳 chunk | 跳 detail.html |

**关键**：权限过滤完全在 RAGFlow 向量库内部完成（`metadata_condition`），不后置、无性能损耗，满足 ≤1s + 文档级权限 + 等保三级。

### 阶段 ⑦ 智能应用

**目标**：把知识能力封装为智能体与对话服务。**管理页**：`agents.html`；**使用页**：`chat.html`。

| 操作 | 页面 | 动作 | RAGFlow API |
|------|------|------|------------|
| 创建智能体 | agents "+ 新建" | 配置 Agent（关联知识库、提示词、工具）| `POST /api/v1/agents` |
| 编辑/删除智能体 | agents 卡片操作 | — | `PUT/DELETE /api/v1/agents/{id}` |
| 创建对话助手 | chat 左侧"助手" | 关联知识库 + LLM | `POST /api/v1/chats`（Create chat assistant）|
| 新建会话 | chat "新建会话" | — | `POST /api/v1/chats/{chat_id}/sessions` |
| 发起对话 | chat 输入框 | 提问 | `POST /api/v1/chats/{chat_id}/completions` |
| 查看溯源 | chat 助手回复下方 | 点击召回 chunk | 同上接口返回的 `reference` 字段 |
| OpenAI 兼容调用 | 外部系统 | 标准格式 | `POST /api/v1/chat/completions`（OpenAI-Compatible）|
| 相关问题推荐 | chat 右侧 | 系统生成 | RAGFlow 相关问题接口 |
| 反馈（赞/踩）| chat 回复操作条 | 评价质量 | `PATCH .../sessions/{session_id}/messages`（feedback）|

**关键**：对话返回的 `reference` 字段含完整召回 chunk（含文档名、相似度、页码），是溯源与质量评估的依据。

## 四、闭环迭代（反馈驱动）

运营阶段的输出反向驱动新一轮采集，形成持续优化闭环：

```
[统计页发现]
  • 高频检索词无命中 → 知识缺口 → 启动 ①采集补充
  • 某 chunk 命中低 → 标注不足 → 回到 ④标注补充关键词
  • 用户反馈答案不准 → 召回质量差 → 回到 ⑤审核/②切分调优
  • 审核退回率高 → 内容质量问题 → 加强 ⑤AI预审规则
  • 权限误拒/误放 → 权限策略漂移 → 修正 ⑧permission 配置
```

**迭代机制**：
1. statistics 页每周生成「知识健康度报告」（覆盖率、命中率、缺口列表）
2. 管理员在 workbench 收到待办，分派给编辑员/知识工程师
3. 编辑员回到阶段①②④补充，重新走完⑤审核发布
4. version 页保留每次迭代的 dataset 快照，支持回滚

## 六、API 闭环速查

|------|-----------------|

## 七、关键约束落地对应

| 强约束 | 在闭环中的落地阶段 | 实现机制 |
|--------|------------------|---------|
| 文档级权限 | ③写入 + ⑥过滤 | metadata + metadata_condition |
| 操作审计 100% | ⑧运营 | 自建 operation_log，全阶段埋点 |
| 等保三级 | ⑧运营 | 权限继承 + 数据分级（security_level）+ 操作快照 |
| 三级审核机制 | ⑤审核 | 模板化状态机 |
| 检索 ≤1s | ⑥检索 | metadata 预过滤 + Redis 权限索引缓存 |
| 知识准确率≥99% | ④⑤⑧ | AI 自动标注 + 三级审核 + 质量评估反馈 |
| 覆盖率 100% | ⑧运营 | 缺口分析反向驱动采集 |
